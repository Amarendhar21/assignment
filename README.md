# README

## Python Library Used

The tool is implemented in Python and uses the following library:

* `easysnmp`

## Installation

Install the required Python package using pip:

```bash
pip install easysnmp
```

## Helper Script

A helper script was used to collect MAC addresses from the switches. The collected MAC addresses were then used to test the `macfind` tool.

The helper script can be executed using:

```bash
python3 snmp_prober.py --hosts <switch_ip> --community <community_string>
```

The script produces a list of learned MAC addresses along with the corresponding switch ports and interface information.

## Running the MAC Finder Tool

Once the MAC addresses have been collected using the helper script, the MAC finder tool can be executed using one or more MAC addresses.

```bash
./macfind <mac1> [mac2 ...] -- <agent1_ip:port:community> [agent2_ip:port:community ...]
```

Supported arguments:

```text
--macs           : One or more MAC addresses to search for
--agents         : One or more SNMP agents in ip:port:community format
--timeout        : SNMP timeout in seconds (default: 5)
--retries        : SNMP retries (default: 2)
--uplink-pattern : Regex pattern to match uplink ports (default: TenGig|Port-channel|Uplink)
```

Example output:

```text
==============================
FINAL REPORT
==============================
aa:bb:cc:dd:ee:ff  |  10.0.0.1  |  GigabitEthernet0/1  |  edge
00:11:22:33:44:55  |  10.0.0.2  |  Port-channel1       |  uplink
de:ad:be:ef:00:00  |  NOT FOUND
```

The output shows the interface associated with the given MAC address. This allows us to identify the corresponding interface name (`ifName`) and determine where the device is connected.

## Verification

To verify the correctness of the results, the output was cross-checked using a standard SNMP walk command.

```bash
snmpwalk -v2c -c public <switch_ip> 1.3.6.1.2.1.2.2.1.2.<ifIndex>
```

The information returned by the SNMP walk was compared with the output generated by the tool to confirm that the correct interface and MAC address mappings were found.
